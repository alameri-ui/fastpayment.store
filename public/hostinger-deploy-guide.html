📤 رفع الملفات إلى السيرفر

الطريقة الأولى (موصى بها): عبر GitHub

إذا رفعت المشروع على GitHub مسبقاً، هذه هي أسهل طريقة:

# تثبيت Git apt install -y git # الانتقال لمجلد التطبيق cd /var/www/falcon-iptv # سحب المشروع (غيّر الرابط بالرابط الخاص بك) git clone https://github.com/اسمك/falcon-iptv.git . # بناء المشروع على السيرفر npm install npm run build
✅ هذه الطريقة تُبقي المشروع محدّثاً — في كل تحديث فقط اكتب: git pull && npm run build

الطريقة الثانية: رفع يدوي عبر FileZilla/WinSCP

  1. افتح WinSCP أو FileZilla
    اختر بروتوكول SFTP
  2. أدخل بيانات الاتصال:
    Host: IP السيرفر | Username: root | Password: كلمة مرور Hostinger | Port: 22
  3. انتقل لمجلد /var/www/falcon-iptv على السيرفر
    من الجانب الأيمن في FileZilla
  4. ارفع هذه الملفات/المجلدات فقط:
    📁 dist/ | 📄 package.json | 📄 drizzle.config.ts | 📁 shared/ | 📁 migrations/ (إن وجد)
  5. بعد الرفع، ثبّت الحزم عبر SSH:
    اكتب في SSH: cd /var/www/falcon-iptv && npm install --production
⚠️ لا ترفع: مجلد node_modules/ (حجمه كبير جداً ويختلف بين الأنظمة) — سيُثبَّت تلقائياً بأمر npm install
الخطوة 5

🗄️ إعداد قاعدة البيانات PostgreSQL

إنشاء قاعدة البيانات والمستخدم

عبر SSH، اتصل بـ PostgreSQL:

# الدخول على PostgreSQL sudo -u postgres psql

بعد الدخول، اكتب هذه الأوامر (غيّر كلمة المرور):

-- إنشاء مستخدم قاعدة البيانات CREATE USER falcon_user WITH PASSWORD 'كلمة_مرور_قوية'; -- إنشاء قاعدة البيانات CREATE DATABASE falcon_iptv OWNER falcon_user; -- إعطاء الصلاحيات GRANT ALL PRIVILEGES ON DATABASE falcon_iptv TO falcon_user; -- الخروج \q

اختبار الاتصال بقاعدة البيانات

psql -U falcon_user -d falcon_iptv -h localhost # أدخل كلمة المرور — إذا دخلت فالاتصال ناجح # اكتب \q للخروج
✅ رابط قاعدة البيانات الذي ستستخدمه:
postgresql://falcon_user:كلمة_مرور_قوية@localhost:5432/falcon_iptv

تطبيق مخطط قاعدة البيانات

بعد إعداد ملف .env (الخطوة التالية)، شغّل هذا الأمر لإنشاء جداول قاعدة البيانات:

cd /var/www/falcon-iptv npm run db:push
💡 npm run db:push يقرأ shared/schema.ts وينشئ جميع الجداول تلقائياً في قاعدة البيانات

⚙️ متغيرات البيئة (.env)

أنشئ ملف .env في مجلد المشروع:

nano /var/www/falcon-iptv/.env

أضف هذا المحتوى (غيّر القيم بما يناسبك):

# ====================================== # قاعدة البيانات — غيّر القيم أدناه # ====================================== DATABASE_URL=postgresql://falcon_user:كلمة_المرور@localhost:5432/falcon_iptv # ====================================== # أمان الجلسات — اجعلها عشوائية وطويلة # ====================================== SESSION_SECRET=أي_نص_عشوائي_طويل_هنا_مثلاً_abc123xyz456def789 # ====================================== # بيئة التشغيل # ====================================== NODE_ENV=production # ====================================== # البورت (اتركه 5000) # ====================================== PORT=5000

احفظ الملف: اضغط Ctrl+O ثم Enter ثم Ctrl+X للخروج

🔒 أمان مهم: لا تشارك ملف .env مع أحد ولا ترفعه على GitHub. يحتوي على كلمات مرور قاعدة البيانات.

توليد SESSION_SECRET عشوائي

يمكنك توليد قيمة عشوائية قوية عبر:

node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

انسخ الناتج وضعه كقيمة لـ SESSION_SECRET

تحميل متغيرات البيئة تلقائياً

تأكد من تثبيت dotenv (مثبت مسبقاً في المشروع):

# التحقق أن المشروع يقرأ .env cat /var/www/falcon-iptv/.env

الخطوة 8

⚡ إعداد PM2 (تشغيل مستمر)

PM2 يضمن أن التطبيق يعمل دائماً — حتى بعد إعادة تشغيل السيرفر

تشغيل التطبيق عبر PM2

cd /var/www/falcon-iptv # تشغيل التطبيق pm2 start dist/index.cjs --name falcon-iptv --env production # حفظ إعدادات PM2 (لتشغيل تلقائي بعد إعادة التشغيل) pm2 save # إعداد PM2 للتشغيل مع النظام pm2 startup # شغّل الأمر الذي يظهره PM2 بعد هذا الأمر

أوامر PM2 المفيدة

الأمرالاستخدام
pm2 statusعرض حالة التطبيقات
pm2 logs falcon-iptvعرض السجلات (مفيد لتتبع الأخطاء)
pm2 restart falcon-iptvإعادة تشغيل التطبيق
pm2 stop falcon-iptvإيقاف التطبيق
pm2 delete falcon-iptvحذف التطبيق من PM2
pm2 monitمراقبة استخدام CPU والذاكرة
✅ الآن التطبيق يعمل على المنفذ 5000 باستمرار. الخطوة التالية ربطه بالدومين عبر Nginx

ملف ecosystem.config.cjs (اختياري - موصى به)

أنشئ هذا الملف لإعدادات PM2 الأكثر تحكماً:

nano /var/www/falcon-iptv/ecosystem.config.cjs
module.exports = { apps: [{ name: 'falcon-iptv', script: 'dist/index.cjs', env_production: { NODE_ENV: 'production', PORT: 5000 }, instances: 1, autorestart: true, watch: false, max_memory_restart: '500M' }] };
# تشغيل باستخدام ملف الإعداد pm2 start ecosystem.config.cjs --env production
الخطوة 9

🌐 إعداد Nginx (ربط الدومين)

📡 قبل هذه الخطوة، اذهب لمزود الدومين (Namecheap, GoDaddy, الخ) وأضف سجل DNS:
A Record: @ → IP السيرفر
انتظر 5-30 دقيقة حتى يتم النشر

إنشاء ملف إعداد Nginx

nano /etc/nginx/sites-available/falcon-iptv

أضف هذا المحتوى (غيّر موقعك.com بدومينك الحقيقي):

server { listen 80; server_name موقعك.com www.موقعك.com; location / { proxy_pass http://localhost:5000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_cache_bypass $http_upgrade; client_max_body_size 50M; } }

تفعيل الإعداد

# ربط الملف بالمجلد النشط ln -s /etc/nginx/sites-available/falcon-iptv /etc/nginx/sites-enabled/ # اختبار صحة الإعداد nginx -t # يجب أن يظهر: syntax is ok + test is successful # إعادة تشغيل Nginx systemctl reload nginx
✅ الآن يمكن الوصول للموقع عبر: http://موقعك.com

🔒 شهادة SSL مجانية (HTTPS)

بعد أن تأكدت أن الموقع يعمل عبر HTTP، ثبّت شهادة SSL مجانية:

تثبيت Certbot

# تثبيت Certbot apt install -y certbot python3-certbot-nginx # طلب شهادة SSL (غيّر موقعك.com) certbot --nginx -d موقعك.com -d www.موقعك.com

اتبع التعليمات:

  1. أدخل بريدك الإلكتروني
    للإشعارات المتعلقة بالشهادة
  2. اضغط A للموافقة على الشروط
  3. اختر: 2 — Redirect (إعادة توجيه HTTP → HTTPS)
    هذا يجعل الموقع آمناً تلقائياً
🎉 الموقع الآن يعمل على HTTPS! الشهادة تتجدد تلقائياً كل 90 يوم

التحقق من التجديد التلقائي

certbot renew --dry-run # يجب أن يظهر: Congratulations, all simulated renewals succeeded
🦅 تهانينا! موقع فالكون IPTV يعمل بالكامل على:
https://موقعك.com

الخطوة 12

🛠️ استكشاف الأخطاء الشائعة

المشكلة السبب الحل
الموقع لا يفتح PM2 متوقف أو Nginx pm2 status و systemctl status nginx
خطأ قاعدة البيانات DATABASE_URL خاطئ تحقق من .env ومن أن PostgreSQL يعمل
502 Bad Gateway التطبيق لا يعمل على 5000 pm2 restart falcon-iptv
خطأ npm install ذاكرة RAM منخفضة أضف Swap: fallocate -l 2G /swapfile
SSL لا يعمل DNS لم ينتشر بعد انتظر 30 دقيقة وأعد المحاولة
db:push يفشل المستخدم لا يملك صلاحيات أعد خطوة إعداد PostgreSQL وتأكد من الصلاحيات

عرض سجلات التطبيق

# سجلات PM2 pm2 logs falcon-iptv --lines 50 # سجلات Nginx tail -f /var/log/nginx/error.log # سجلات النظام journalctl -u nginx -n 50

إضافة ذاكرة مؤقتة (Swap) إذا كانت الـ RAM 1GB

fallocate -l 2G /swapfile chmod 600 /swapfile mkswap /swapfile swapon /swapfile echo '/swapfile none swap sw 0 0' | tee -a /etc/fstab
